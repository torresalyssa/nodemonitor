Install

This will check whether or not you have PM2 (the tool used to report monitoring information on your project), your project, Node.js, npm, and bower installed. If any of these are not detected, you will need to click the corresponding button(s) to install them. This page will offer to install your project from the git_repo listed in your config.json file if there is no local install.

If you get an error message while trying to install, make sure that you have admin privileges.

Make sure there is a check mark next to each dependency before proceeding with the use of this app.


Basic Dashboard

The first item listed on the basic dashboard indicates your project and its status (running or not running). When your project is not running, you can try to start it by clicking the start icon. When your project is running, you can stop it or restart it by clicking the icons corresponding to those actions.

The second item listed on the basic dashboard represents the watchdog (PM2) that is monitoring your app. The icon to the left indicates whether or not the watchdog is active (whether or not PM2 is reporting monitoring information to the PM2 endpoint given in the config.json file).


Advanced Dashboard

The advanced dashboard allows you to view more detailed information on the status of your app, including a list of all running processes, the ability to start/stop/reset any process, and CPU and memory usage information.


Update

Click the Check for Updates button to see if your git project is up-to-date. Click the Update button to update your local version of the git project located at the git project path (project_path) given in the configuration. The update process will pull the latest version of the project and run npm update's and bower update's as needed.


Configuration File (config.json)

ping_endpt:  Endpoint for your project. Used to ping and determine if the project is running.
project_name:  The name of your project. Used only for UI purposes.
pm2_endpt:  Where PM2 is reporting its monitoring information to.
project_path:  Absolute path to your project. If this is left blank/empty, then it is assumed that there is no local install of the project and you will need to go to the Install page to clone the project from the given git_repo.
main_project_file:  File to be started via pm2 start in order to run your project. Must be located in the root of the project directory.
git_repo:  The location of the git repository for this project. Used to install the project if there is no local install, i.e. there is no project_path.

Troubleshooting

Issue: I got an error when trying to install PM2/Bower.
Solution: This most likely means that you are not running with administrative privileges. Make sure you are logged in to your computer as an administrator and try again. If that still does not work, make sure that you are connected to the internet.

Issue: The watchdog is indicating that it is not running.
Solution: This could mean that you have the wrong PM2 endpoint in your config.json file. Check this by making sure that the port number output by PM2 when you run pm2 web in the terminal matches that given in your config.json under pm2_endpt. Another reason this may occur is that you do not have PM2 installed. Check the Install page to see if you have PM2 installed, and if not click the button to install it.

Issue: It says "Error reading configuration file."
Solution: This usually means that you either do not have a config.json file located in the root of this app, or if you do that it is ill-formed or empty. To fix this, make sure you have a config.json file in the root directory of this app and that it has the format described above.

Issue: I received an error while checking for updates.
Solution: This most likely means that the project indicated by your project_path does not have a git repository. Make sure that your project has a git repository and that the project_path (visible from the App Configuration page) is correct.

Issue: I received an error while updating.
Solution: One reason this could happen is if the project_path is not an absolute path to your project. Make sure the project path is NOT a relative path by looking at the Configuration page. An error while updating could also mean that you do not have Bower and/or npm installed. Check the install page to verify they are installed, and install them if they are not.

Issue: An install button is disabled.
Solution: First, refresh the page. Any buttons that are disabled and without a check mark next to them are dependencies that cannot be installed until some other dependency is installed. Make sure that you click on the non-disabled buttons to install those dependencies first, and once that is done you will be able to install the others.
