Install

This will check whether or not you have PM2 (the tool used to report monitoring information on your project) installed. If it is not detected on your computer, a button will show up that you can click to install PM2, which is necessary for this app to be able to supply you with monitoring information. Additionally, this page will offer to install your project from git if there is no local install.

If you get an error message while trying to install PM2, make sure that you have permission to install programs globally on your computer.


Basic Dashboard

The first item listed on the basic dashboard indicates your project and its status (running or not running). When your project is not running, you can try to start it by clicking the start icon. When your project is running, you can stop it or restart it by clicking the icons corresponding to those actions.

The second item listed on the basic dashboard represents the watchdog (PM2) that is monitoring your app. The icon to the left indicates whether or not the watchdog is active (whether or not PM2 is reporting monitoring information to the PM2 endpoint given in the config.json file).


Advanced Dashboard

The advanced dashboard allows you to view more detailed information on the status of your app, including a list of all running processes, the ability to start/stop/reset any process, and CPU and memory usage information.


Update

Click the Check for Updates button to see if your git project is up-to-date. Click the update button to update your local version of the git project located at the git project path (project_path) given in the config.json file. The update process will pull the latest version of the project and run npm update's and bower update's as needed.


Configuration File (config.json)

ping_endpt:  Endpoint for your project. Used to ping and determine if the project is running.
project_name:  The name of your project.
pm2_endpt:  Where PM2 is reporting its monitoring information to.
project_path:  Local path to your project.
main_project_file:  File to be started via pm2 start in order to run your project. Must be located in the root project directory given by project_path. git_repo:  The location of the git repository for this project. Used to install the project if there is no local install, i.e. there is no project_path.

Troubleshooting

Issue: The watchdog is indicating that it is not running.
Solution: This could mean that you did not tell PM2 to start reporting data to the PM2 endpoint. Do this by typing the command pm2 web into your terminal. Another reason the watchdog could not be running is that you have the wrong PM2 endpoint in your config.json file. Check this by making sure that the port number output by PM2 when you run pm2 web matches that given in your config.json under pm2_endpt.

Issue: The watchdog is indicating that it is running, but my app is indicating it is not.
Solution: This means that you have PM2 running a different app and reporting monitoring information on that app to the PM2 endpoint. To fix this, run pm2 kill in your terminal, navigate to your project's main .js file, run pm2 start [main app file], and finally run pm2 web to have PM2 start reporting to the PM2 endpoint.
